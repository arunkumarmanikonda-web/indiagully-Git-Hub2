| Portal | URL | Authentication |
|--------|-----|---------------|
| Super Admin | `/admin` | Admin username + password + **RFC 6238 TOTP** |
| Client | `/portal/client` | Client email + password + **RFC 6238 TOTP** |
| Employee | `/portal/employee` | Employee ID + password + **RFC 6238 TOTP** |
| Board & KMP | `/portal/board` | Director DIN/KMP ID + password + **RFC 6238 TOTP** |

## 🔒 H-Round Security Hardening (LIVE ✅ — 2026-03-01)

**Tag:** `v2026.06-H` · **Security Score:** 78/100 ↑ (was 72/100 in G-Round)

| ID | Severity | Finding | Fix Applied | Status |
|----|----------|---------|-------------|--------|
| **H1** | CRITICAL | TOTP RFC 6238 Base32 bug — server `computeHOTP()` used `TextEncoder` (raw UTF-8 bytes) instead of proper Base32 decode; codes never matched any RFC 6238 authenticator app (Google Authenticator, Authy, etc.) | Added `base32Decode()` function to `api.tsx computeHOTP()`; updated client-side `igFillTOTP()` in admin login with matching decode | ✅ RESOLVED |
| **H2** | HIGH | No server-side session guard on admin and portal sub-routes — all dashboard pages accessible without authentication | Added `app.use('/*')` middleware in `admin.tsx` and `portal.tsx`; unauthenticated requests redirect to login; public paths whitelisted | ✅ RESOLVED |
| **H3** | MEDIUM | Admin portal pages fully static — Finance/HR/Governance/Contracts/BI/Security/KPI/Risk/Integrations showed hardcoded data; all actions were `igToast()` stubs | Added `window.igApi` fetch client to `adminShell`; wired all admin pages to real API endpoints; `igCreateInvoice()` → `POST /api/finance/voucher`; `igProcessPayroll()` → `POST /api/hr/payroll/run`; `igGenIRN()` → `POST /api/finance/einvoice/generate`; Sales dashboard → `/api/mandates` + `/api/sales/commission/summary` | ✅ RESOLVED |
| **H4** | CRITICAL | Admin login TOTP auto-fill also used `TextEncoder` — codes generated by browser never matched server (or authenticator apps) | Replaced `TextEncoder` with proper Base32 decode in client-side `computeHOTP()` in admin login page | ✅ RESOLVED |

## 🏗️ Architecture

- **Platform:** Cloudflare Pages / Workers (edge runtime, 120+ routes)
- **Framework:** Hono v4.12 (TypeScript)
- **Frontend:** Server-side HTML + Tailwind CSS CDN + FontAwesome + Chart.js
- **Auth:** PBKDF2-SHA256 + RFC 6238 TOTP + HttpOnly session cookie + CSRF synchronizer token
- **Storage:** Cloudflare D1 (schema ready) · R2 (documents) · KV (sessions, rate-limiting)
- **Headers:** HSTS, X-Frame-Options (DENY), X-Content-Type-Options, Referrer-Policy, CSP, Permissions-Policy
- **CI/CD:** GitHub Actions (build + type-check + gitleaks scan + CF Pages deploy + smoke test)
